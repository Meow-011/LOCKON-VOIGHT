# Security Policy

## Supported Versions

Currently, only the latest release of LOCKON VOIGHT is officially supported for security updates. 

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

As LOCKON VOIGHT is an anti-cheat and integrity monitoring system, we take security vulnerabilities extremely seriously. 

**DO NOT report security vulnerabilities via public GitHub Issues.**

If you discover a security vulnerability (e.g., a bypass in the eBPF module, a JWT authentication flaw, or an API exploit), please report it by **opening a private GitHub Security Advisory**. 

To do this, go to the **Security** tab of this repository, select **Advisories**, and click **Report a vulnerability**.

### What to include:
- A detailed description of the vulnerability.
- Steps to reproduce the exploit.
- Potential impact (e.g., "Allows a contestant to spoof their hardware identity").

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. 

Thank you for helping keep competitive CTF environments fair and secure!
